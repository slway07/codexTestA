# Tarot MVP (No-LLM First)

이 저장소는 **LLM 없이 시작하는 타로 웹사이트 MVP**를 구현합니다.

## 목표
- 핵심 사용자 흐름(질문 → 카드 뽑기 → 해석 보기)을 빠르게 검증한다.
- 초기에는 룰/템플릿 기반 해석을 사용하여 비용과 복잡도를 낮춘다.
- 이후 단계적으로 LLM을 붙여 답변 품질과 개인화를 고도화한다.

## 현재 구현 범위 (MVP v1)
- 질문 입력(선택)
- 질문 카테고리 선택(연애/일·커리어/금전/일반)
- 메이저 아르카나 22장 중 1장 랜덤 뽑기
- 정/역방향 랜덤 결정
- 카드 기본 키워드 + 카테고리별 템플릿 해석 + 질문 반영 문구
- 최근 리딩 기록 10개 로컬 저장(localStorage)

## 실행 방법
정적 파일만 사용하므로 브라우저에서 `index.html`을 열면 실행됩니다.

간단히 로컬 서버를 띄우려면:

```bash
python3 -m http.server 4173
```

그 다음 브라우저에서 `http://localhost:4173` 접속.

## 제품 로드맵

### Phase 1 — MVP (현재)
- 룰/템플릿 기반 해석
- 최소 UX 검증
- 기본 지표 수집 준비

### Phase 2 — MVP+ (LLM 제한적 도입)
- "AI로 자세히 설명" 버튼 추가 (선택 호출)
- 카드 데이터(JSON) 범위 안에서만 답변하도록 프롬프트 가드레일
- 답변 길이/호출 횟수 제한으로 비용 통제

### Phase 3 — 고도화
- 3카드 스프레드(과거/현재/미래)
- 사용자 계정 및 리딩 히스토리 서버 저장
- 개인화 톤, 재방문 리마인드, A/B 테스트

## 왜 No-LLM First인가?
1. **빠른 검증**: 핵심 가치(타로 경험 자체)부터 확인 가능
2. **낮은 리스크**: 환각/안전성/비용 문제 최소화
3. **유연한 확장**: LLM을 선택적으로 붙여 점진적 개선 가능

## LLM 도입 시 체크리스트
- 카드/해석 데이터 외 근거 없는 답변 금지
- 의료/법률/투자 단정형 문구 금지
- 민감정보 수집 최소화
- 호출 비용/지연 모니터링
